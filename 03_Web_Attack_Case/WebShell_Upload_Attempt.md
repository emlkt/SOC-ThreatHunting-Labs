# Попытка загрузки Web Shell и успешное предотвращение через EDR

## Гипотеза:
**Злоумышленники могут использовать функцию загрузки файлов в веб-приложении для повреждения сервера или создания бэкдоров путем загрузки вредоносных файлов.**

Основанием послужили алерты WAF (FortiWeb) о подозрительной активности на эндпоинте `/upload.aspx` и события EDR (SentinelOne) на веб-сервере.

## Резюме инцидента:
В ходе расследования мной была выявлена многоэтапная атака, которая была направлена на загрузку веб-шелла `backdshell.aspx` на корпоративный сервер под управлением IIS. Злоумышленник использовал несколько IP-адресов для разведки и загрузки полезной нагрузки. Несмотря на то, что WAF обнаружил атаку, но пропустил трафик (вероятно режим мониторинга), система endpoint-защиты (SentinelOne) успешно детектировала вредоносный файл, уничтожила его и поместила в карантин. Попытка злоумышленника запустить шелл завершилась ошибкой **404 Not Found**, так как файл уже был удален с диска.

Данный нцидент можем считать предотвращенным, но требуется настройка WAF на блокировку.

## Логика расследования

### Шаг 1: Пыталась выявить любые аномалии в веб-трафике
Начала с анализа логов доступа IIS, чтобы найти наиболее активные запросы к форме загрузки.
*   `rule.groups: "web, accesslog" + data.url: "/upload.aspx"` и др.
*   Увидела что запросы идут от совершенно разных устройств из разных стран за короткое время: `iPhone`, `Linux`, `Windows`, `Android`, что явный признак ботнета.
*   Обнаружила, что генерировал наибольшее количество запросов генерировал ip `122.171.207.xxx`.

### Шаг 2: Проверила, как межсетевой экран реагировал на эти запросы
*   `rule.groups: "waf" + data.url: "/upload.aspx"`
*   Тот же IP `122...` и другой ip `37.77.88.xxx` вызывали алерты категории `Web Shell Attack`.
*   Действие WAF было `passthrough`. Скорее всего правило работает в режиме мониторинга (alert-only). Далее нужно было проверить, достиг ли файл диска

### Шаг 3: Проверила состояние хоста через логи EDR
Обратилась к логам SentinelOne, чтобы узнать итоги загрузки файла.
*   Искала по пути `uploads` и различным событиям, связанным с `SentinelOne`.
*   Нашла загруженный файл `backdshell.aspx` со статусом действия: `kill-quarantine`.
*   Файл был вредоносным веб-шеллом, но антивирус успел его обезвредить.

### Шаг 4: Далее необходимо подтверждение неудачи атаки
Чтобы убедиться, что код не был выполнен, проверила логи IIS на наличие GET запросов к этому файлу.
*   `data.url: "/uploads/backdshell.aspx"`
*   Запрос от IP `145.13.10.xxx` к `backdshell.aspx` вернул статус **404**.
*   Атакующий попытался запустить шелл, но получил ошибку «Файл не найден», так как EDR уже удалил его. Цепочка атаки разорвана.

## Детальный анализ событий

### 1. Разведка и загрузка (логи WAF)
**География: IP из Азии, Европы, Африки, США.**
* **User-Agent** - рандомизация между мобильными и десктопными браузерами (в т.ч. устаревших версий):
* `Apple-iPhone13C2`
* `Linux x86_64 (Chrome)`
* `Android 9 (Pixel 3)`
*  `и др.`
  + нашла `Mozilla/5.0 AppleWebKit/537.36 (KHTML, like Gecko; compatible; GPTBot/1.2; +https://openai.com/gptbot)` - возможно это попытка выдать себя за сервисы искусственного интеллекта

WAF корректно классифицировал атаку, но настройки политики не позволили блокировать трафик.
```json
{
  "subtype": "waf",
  "srcip": "37.77.88.xxx",
  "url": "/upload.aspx",
  "method": "POST",
  "category": "Web Shell Attack",
  "action": "passthrough", 
  "filename": "backdshell",
  "filetype": "aspx"
}
```
Категория `Web Shell Attack` подтверждает то, что файл вредоносный. Действие `passthrough` указывает на режим мониторинга правила, что создало кратковременное окно уязвимости.

### 2. Нейтрализация угрозы (логи SentinelOne )
EDR удалил вредоносный файл
```json
{
  "event_type": "threat_detected",
  "product": "SentinelOne",
  "action_taken": "kill-quarantine",
  "threat_name": "Web Shell Detedted",
  "process_path": "C:\\inetpub\\w3svc01\\uploads\\backdshell.aspx",
  "status": "resolved"
}
```

### 3. Попытка запуска (логи IIS)
Тут хакер пытался активировать шелл
```text
Date: 2024-08-19 05:02:14
SrcIP: 145.13.10.xxx
Method: GET
URI: /uploads/backdshell.aspx
Status: 404
```

##  MITRE ATT&CK
*   **T1190 (Exploit Public-Facing Application):** Эксплуатация формы загрузки.
*   **T1505.003 (Server Software Component: Web Shell):** Попытка установки веб-шелла.
*   **T1059 (Command and Scripting Interpreter):** Планируемое выполнение команд.
*   **T1071 (Application Layer Protocol):** Использование HTTP/HTTPS для команд и управления (ботнет).
