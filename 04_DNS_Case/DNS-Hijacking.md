# Обнаружение DNS Hijacking и эксплуатации уязвимости SigRed (CVE-2020-1350)

## Гипотеза
**Злоумышленники могут манипулировать DNS-запросами в корпоративной сети, осуществляя атаки DNS Hijacking для перенаправления пользователей на поддельные веб-сайты с целью кражи учетных данных или доставки вредоносного ПО.**

## Резюме инцидента
Во время расследования в Wazuh я выявила многоэтапную атаку, начавшуюся с эксплуатации критической уязвимости CVE-2020-1350 на DNS-сервере компании. Злоумышленник, идентифицированный как группа APT-M070 (по информации из внутреннего TI), использовал уязвимость удаленного выполнения кода (RCE) для получения контроля над сервером. После компрометации была удалена легитимная DNS-запись типа **A** для внутреннего ресурса `ghsprofessional.company.local` ip `10.0.0[.]125` и создана новая запись с тем же именем, но указывающая на внешний вредоносный ip `147[.]23[.]7[.]30`. Это классический признак DNS Hijacking. Несмотря на то, что фаервол заблокировал попытки нескольких пользователей подключения к новому ip, факт модификации DNS присутствует и это свидетельствует о серьезной компрометации инфраструктуры.
*APT-M070 - существует в рамках лабораторной работы.*

## Логика расследования

### Шаг 1: Выявление критической уязвимости
Начала с анализа отчетов сканера уязвимостей Nessus в Wazuh, чтобы найти слабые места в инфраструктуре.
* `rule.groups: "nessus"`, `data.severity: "critical"`.
* Обнаружено событие: **Microsoft DNS Server RCE Vulnerability Detected** c CVE-2020-1350.
* Цель - Windows Server 2012 R2 (DNS Server). CVSS score 10.0.

### Шаг 2: Детекция эксплуатации через IDS/IPS
Проверила, были ли попытки эксплуатации найденной уязвимости, анализируя логи ids Suricata.
* `rule.groups: "ids" AND rule.groups: "suricata" AND event.type: "alert"`.
* Было найдено событие с сигнатурой: **ET EXPLOIT Microsoft Windows DNS Server SIG Record Parsing RCE Attempt (CVE-2020-1350)**.
* Source IP атакующего - `92[.]8[.]100[.]23`

### Шаг 3: Поиск в Threat Intelligence 
Проверила ip-адрес атакующего `92[.]8[.]100[.]23` в базе Threat Intelligence (LetsDefend CTI).
* TI показал, что ip ассоциирован с группой APT-M070, значит атака проводится продвинутой группой угроз, использующей известный эксплойт для первоначального доступа.

### Шаг 4: Анализ изменений в DNS 
Изучила аудиторские логи Windows DNS Server, чтобы понять действия злоумышленника после получения доступа.
* Нашла событие удаления с Event ID 513:
    * Удалена запись: `ghsprofessional.company.local`.
    * Тип: A
    * Старый IP (RDATA) `10.0.0[.]125`.
* После нашла событие добавления с Event ID 515:
    * Добавлена запись с тем же именем: `ghsprofessional.company.local`.
    * Тип: A
    * Новый ip (RDATA): `147[.]23[.]7[.]30` (внешний ip).
* (Пришлось немного поискать информацию по этим событиям, но мне помогло: https://learn.microsoft.com/ru-ru/windows-server/networking/dns/dns-logging-and-diagnostics?tabs=desktop-experience)
* В итоге злоумышленник подменил внутренний адрес на внешний, чтобы перенаправлять трафик пользователей на свой сервер (что является примером DNS Hijacking), пользователи продолжают обращаться к привычному внутреннему ресурсу, но DNS-сервер теперь возвращает им адрес внешнего сервера хакера. 

### Шаг 5: Анализ сетевого трафика и реакции Firewall
Ппроверила, пытались ли пользователи подключиться к новому вредоносному ip (`147[.]23[.]7[.]30`), и как отреагировал фаервол.
* `rule.groups: "firewall" AND data.dstip: "147[.]23[.]7[.]30"`.
* Было обнаружено огромное количество взаимодействий с этим ip от 4 различных внутренних IP-адресов.
* Действие фаервола - `deny`.
* В итоге пользователи пытались обратиться к подмененному ресурсу, но межсетевой экран предотвратил соединение.

## Более детальный анализ артефактов

### 1. Уязвимость и Эксплуатация (логи Nessus & Suricata)
Эти логи показывают наличие дыры и попытку её использования.
```json
{
  "event_type": "vulnerability",
  "data": {
    "cve": "CVE-2020-1350",
    "severity": "critical",
    "description": "Microsoft DNS Server is vulnerable to remote code execution via a crafted SIG record."
  }
}
```
```json
{
  "event_type": "alert",
  "alert": {
    "signature": "ET EXPLOIT Microsoft Windows DNS Server SIG Record Parsing RCE Attempt (CVE-2020-1350)",
    "src_ip": "92[.]8[.]100[.]23"
  }
}
```
> Использование CVE-2020-1350 позволяет выполнить код с правами SYSTEM на DNS-сервере без аутентификации.

### 2. Манипуляции с DNS записями (Windows Audit)
Логи демонстрируют удаление легитимной записи и создание вредоносной.
```json
{
  "event_id": 513,
  "message": "A resource record of type 1, name ghsprofessional.company.local... was deleted",
  "rdata": "10.0.0[.]125"
}
```
```json
{
  "event_id": 515,
  "message": "A resource record of type 1, name ghsprofessional.company.local... was created",
  "rdata": "147[.]23[.]7[.]30"
}
```
> Подмена IP адреса при сохранении имени домена это ключевой признак DNS Hijacking. Пользователи не замечают подмены, так как вводят привычное имя.

### 3. Сетевая активность и блокировка фаерволом
Попытки подключения клиентов к вредоносному IP.
```json
{
  "type": "traffic",
  "srcip": "10.0.0[.]63",
  "dstip": "147[.]23[.]7[.]30",
  "service": "HTTPS",
  "action": "deny"
}
```
> Фаервол успешно заблокировал трафик, но 4 хоста уже попытались установить соединение, что требует их проверки на наличие компрометации.

## MITRE ATT&CK
* **T1190 (Exploit Public-Facing Application):** Эксплуатация уязвимости в публичном сервисе (DNS).
* **T1574.006 (Hijack Execution Flow: DNS Hijacking):** Перенаправление трафика через модификацию DNS записей.
* **T1059 (Command and Scripting Interpreter):** Выполнение команд через RCE.
* **T1048 (Exfiltration Over Alternative Protocol):** Потенциальная кража данных через установленный контроль.
