# Атака программы вымогателя через зараженный USB и латеральное перемещение

## Гипотеза
**Сотрудник мог подключить зараженный USB-накопитель к корпоративному устройству, что привело к запуску программы-вымогателя и шифрованию данных.**

Основанием стали алерты от системы Endpoint Protection (EPP) SentinelOne о детектировании активности шифровальщика. Так же эта гипотеза подтвердилась и расширилась: обнаружились логи скрытия следов и попытки распространения на другие узлы сети.

**Статус:** **Критический.** Зафиксировано шифрование файлов, создание требования выкупа, установка инструментов удаленного доступа и анонимизации, а также перемещение носителя на второй хост.

## MITRE ATT&CK
*   **T1091 (Move Media via Removable Media):** Использование USB носителя для переноса малваря между хостами.
*   **T1486 (Data Encrypted for Impact):** Шифрование файлов с целью вымогательства.
*   **T1204.001 (User Execution):** Запуск вредоносного файла пользователем.
*   **T1071.001 (Web Protocols):** Попытка связи через Tor Browser.
*   **T1219 (Remote Access Software):** Использование AnyDesk как для закрепления, так и для удаленного доступа.

## Ход расследования

## Детектирование угрозы
#### 1. Использован фильтр для отбора событий от агента SentinelOne:
`rule.groups: "sentinelone" AND data.threat_type: "ransomware"`

Система зафиксировала активность шифровальщика на хосте `10.0.12.xxx` (HR-WS-124):
*   **Угроза:** `Ransomware.Win32.Locky`.
*   **Действие EPP:** `Detected`.
*   **Вредоносный процесс:** `encryptor.exe`.
*   **Путь его запуска:** `F:\data\j.doe\AppData\Roaming\encryptor.exe`.
    + запуск произошел непосредственно с диска `F:`, определенного как USB.
*   **Хеш (sha256):** `1a2b3c...7890`.

#### 2. Анализ событий подключения устройств подтвердил использование съемного носителя.
`agent.ip: "10.0.12.xxx" AND data.activity_type: "drive_mount" AND data.mount_type: "removable_storage"`
**Было обнаружено:**
*   **Устройство:** Диск `F:` (SanDisk, серийный номер `SN1234567890`).
*   **Время подключения:** 2024-08-22 14:12:54.
*   **Пользователь:** `j.doe`.
*   Скорее всего, сработал автозапуск или пользователь случайно запустил исполняемый файл, маскирующийся под документ.

#### 3. Последствия шифрования
Вредоносный процесс зашифровал большое количество критически важных документов документов:
*   Целевое расширение: `.docx`.
*   Директории `Documents`, `Desktop`, `Shared`, `Backups` и др. на дисках C, D, E.
*   В каждой пораженной папке был создан файл `README.TXT` (скорее всего чтобы его быстрее заметили)

**Пример одного из логов шифрования:**
```json
{
  "timestamp": "2024-08-22T14:47:11.729Z",
  "event_type": "ransomware_activity_detected",
  "threat_name": "Ransomware.Win32.Locky",
  "encrypted_files": [
    "C:\\Users\\j.doe\\Documents\\Financial_Statement.docx",
    "C:\\Shared\\Legal\\Contract_Template.docx"
  ],
  "message": "Ransomware - Encrypt File Activity Detected"
}
```
#### 4. По изучав логи с помощью событий + View  surrounding documents, нашла два критических артефакта, указывающих на сложность атаки:

 а. **Tor Browser** 
- EPP SentinelOne зафиксировал запуск процесса `firefox.exe` из нестандартного пути: `C:\Users\j.doe\AppData\Roaming\Tor Browser\Browser\` практически одновременно с началом шифрования и с алертом:  `TorBrowser.MaliciousActivity` (Severity: High).
- Но в доступных логах сетевое соединение не было явно зафиксировано. Это может означать, что или EPP заблокировал процесс на этапе запуска, не дав установить соединение или соединение планировалось, но было прервано на ранней стадии.

 б. **AnyDesk** 
- Через ~3 часа после начала атаки  была установка приложения `AnyDesk Remote Desktop`. В логах нет данных о скачивании файла из интернета в момент установки + канал связи (Tor) еще не был подтвержден.
- Скорее всего с разницей во времени связан  **скрипт с отложенным выполнением**, который находился на той же зараженной флешке.

 #### 5. Латеральное перемещение
- В логах зафиксировано подключение устройства с идентичным серийным номером (`SN1234567890`) и вендором.
- Хост `10.1.55.xxx` и пользователь `a.smith`.
- Во время анализа активных алертов на шифрование не обнаружено (вероятно - ограничение в сценарии лабораторной работы)
