# Расследование инцидента: Атака "Golden Ticket" на контроллер домена (LetsDefend)

**Дата прохождения:** 16.03.2025  
**Ссылка на задание:** [Golden Ticket Challenge](https://app.letsdefend.io/challenge/golden-ticket)

---

## Описание инцидента

В корпоративной сети сработало оповещение о возможной атаке на **Контроллер Домена (DC)**. Система безопасности зафиксировала подозрительную активность, указывающую на **lateral movement** (перемещение внутри сети) с скомпрометированной рабочей станции.

## Использованные инструменты и источники

1. **Event Log Explorer** для анализа файла `.evtx` и фильтрации событий.
2. **MITRE ATT&CK Framework**, где изучала контекст тактик: T1558.004 (AS-REP Roasting), T1558.003 (Golden Ticket)

---

### 1. Доступ к учетной записи службы (вопросы 1-3)

1. Открыла журнал `Security.evtx` в Event Log Explorer.
2. Применила фильтр: **Event ID 4624**.
3. Добавила пользовательский фильтр по описанию в description params
   - Name: `New Logon\Account Name`
   - Operator: `contains`
   - Value: `service` (предположительно,что название сервисной службы будет его содержать)
1. Поискала события по типу входа 3 - сетевой вход.

#### Было найдено событие:
```
Event ID: 4624
Account Name: SQLService
Account Domain: SOPRANOS
Logon Type: 3
Source IP: 192.168.110.129:48858
Authentication: NTLM
Time: 2024-10-05 16:50:29 UTC
```
(для сервисных аккаунтов в домене нормой является Kerberos, поэтому вход через устаревший NTLM может указывать на атаку pass-the-hash или несанкционированный доступ)

#### Ответы:
1. Время первого доступа к сервисному аккаунту - `2024-10-05 16:50:29 UTC`
2. Имя скомпрометированного аккаунта - `sqlservice`
3. IP и порт атакующего - `192.168.110.129:48858`

---

### 2.Обнаружение AS-REP Roasting (вопросы 4-5)

1. Почитала про стратегию обнаружения в [MITRE ATT&CK: AS-REP Roasting](https://attack.mitre.org/techniques/T1558/004/) и нашла важные индикаторы:
   - **Event 4768** Kerberos TGT request с PreAuthType = 0 (отключена предварительная аутентификация)
   - **Event 4769** Service Ticket request с TicketEncryptionType = 0x17 (слабое шифрование RC4)
2. Поиск по фильтрам:
   - `Event ID 4768` + `PreAuthType = 0`
   - `Event ID 4769` + `TicketEncryptionType = 0x17`

#### Найденное событие:
```
Event ID: 4768
TargetUserName: Corrado
PreAuthType: 0 (NO PREAUTH)
Time: 2024-10-05 14:42:44 UTC
```

(на учетной записи Corrado была включенна опция "Do not require Kerberos preauthentication", а это дает возможность запросить зашифрованный тикет без знания пароля и подобрать его офлайн)

####  Ответы:
4. Целевая учетная запись для AS-REP атаки - `corrado` |
5. Время запроса TGT для атаки - `2024-10-05 14:42:44 UTC`

---

### 3. Поиск событий Golden Ticket (вопросы 6-7)

1. Чтобы найти целевуюучетную запись, исключила:
   - Учетнаю запись `Corrado`
   - Компьютерный аккаунт `SOPRANOS-DC$`
2. Применила фильтр по событиям:
   - `4768` TGT запрос, `4769` сервисный тикет, `4624` успешный вход и `4625` неудачный вход и текст в описании: `Administrator`

 Была найдена мгновенная последовательность 4768+4769+4624 (без задержки на реальную аутентификацию) с использованием NTLM вместо Kerberos и вход под `Administrator`.
#### Ответы:
6. Целевая учетная запись для Golden Ticket - `administrator`
7. Время входа с поддельным тикетом - `2024-10-05 17:57:03 UTC`

**Результат атаки:** Успешный вход под `Administrator` в 17:57:03 через Golden Ticket (подтверждено событием 4624)
